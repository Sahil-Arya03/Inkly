## 1. System Overview

**Inkly** is a web-based team workspace application centered on a kanban board. It is structured as a **classic two-tier SPA + REST API** architecture:

```
Browser (React 18, no build step)
        │  HTTP / cookie
        ▼
Spring Boot 4.1 REST API  (:8081)
        │  JDBC / JPA
        ▼
PostgreSQL (Neon cloud, ap-southeast-1)
```

The project started as a note-taking app (`NoteTake` artifact ID) and evolved into a full workspace tool. The rename to "Inkly" is only in the UI and domain — the Maven artifact, Java package roots, and WAR filename remain `NoteTake`.

### 2.1 Frontend

```
Inkly.html  ← single HTML entry point
    │
    ├── CDN scripts (React 18, ReactDOM, React Router v6, Babel Standalone 7.29)
    │

    ├── inkly-icons.jsx     Icons namespace (SVG components)
    ├── inkly-data.jsx      window.INKLY_DATA — mock data constants
    ├── inkly-chrome.jsx    Sidebar + Topbar layout shell
    ├── inkly-login.jsx     Login component
    ├── inkly-signup.jsx    Signup component
    ├── inkly-dashboard.jsx Dashboard page
    ├── inkly-kanban.jsx    Kanban board (live)
    ├── inkly-tasks.jsx     My Tasks page (mock)
    ├── inkly-calendar.jsx  Calendar page (mock)
    ├── inkly-teams.jsx     Teams page (mock)
    ├── inkly-inbox.jsx     Inbox page (mock)
    ├── inkly-settings.jsx  Settings page (mock)
    └── inkly-app.jsx       App root, router, context, route guards
```

**Key constraints:**
- No build step — Babel transpiles JSX in the browser at runtime (development-only approach).
- All files are loaded as `<script type="text/babel">` in a fixed dependency order.
- Script load order is a hard constraint — inserting a file in the wrong position breaks the app.
- React Router v6 with `HashRouter` (URLs use `#/route`).
- Global state via `InklyContext` (React context), no external state library.

### 2.3 Authentication Flow

```
[Browser]                          [Spring Boot]
    │
    │  POST /api/auth/login
    │  { email, password, rememberMe }
    │ ─────────────────────────────────►
    │                                  1. Validate credentials vs APP_USERS (BCrypt)
    │                                  2. WorkspaceSetupService.bootstrap() (idempotent)
    │                                  3. JwtUtil.generate(email, rememberMe)
    │                                  4. Set-Cookie: inkly_token=<JWT>; HttpOnly; SameSite=Lax
    │  ◄────────────────────────────────
    │  AuthResponse { email, name, workspace, expiresIn }
    │  (JWT NOT in body — HttpOnly only)
    │
    │  GET /api/me  (cookie sent automatically)
    │ ─────────────────────────────────►
    │                                  JwtAuthFilter reads inkly_token cookie
    │                                  → JwtUtil.isValid() → extractEmail()
    │                                  → UserDetailsServiceImpl → SecurityContext
    │  ◄────────────────────────────────
    │  { email, name, workspace, role }
    │
    │  (authenticated API calls use cookie automatically — credentials:'include')
```

### 2.4 Domain Model (abbreviated)

```
APP_USERS (auth)
    │ email (shared key, not FK)
    ▼
KanbanUser (users table) ────────────► Membership ────────► Workspace
    │                                        │                    │
    │                                    Department          WorkspaceCounter
    │                                                             │
    └── Card.createdBy                   Board ───────► BoardColumn
            │                               │
            └── CardAssignee           Sprint
                                            │
                                         Card ──────► CardLabels → Label
                                            │
                                         Comment
                                         Attachment
```

### 2.5 Data Flow — Creating a Card

```
Browser
  1. User fills CreateTaskModal (title + description required)
  2. INKLY_API.kanban.createCard({ workspaceId, boardId, columnId, title, description, priority, assigneeId? })
  3. POST /api/cards  [cookie: inkly_token]
  4.   KanbanController.createCard(req, auth.getName())
  5.   CardService.createCard():
        a. Resolve Workspace, Board, BoardColumn, KanbanUser
        b. SELECT … FOR UPDATE on workspace_counters (pessimistic lock)
        c. Claim next seq → increment counter
        d. Compute rank via RankGenerator.between(afterRank, null)
        e. INSERT into cards
        f. Optional: validate assignee → INSERT into card_assignees
        g. Return CardResponse
  6. Browser optimistically adds card to state
```
